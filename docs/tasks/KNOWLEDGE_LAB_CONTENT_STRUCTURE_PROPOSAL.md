# Knowledge Lab: 內容結構提案與初學者友善化 (Proposal)

為了提升 Lingourmet 知識文庫（Knowledge Lab）對初學者的友善程度，同時對齊核心 content-ko 的解耦架構，本文件提案一套標準化的內容編寫風格與其對應的 UI 呈現邏輯。

## 1. 核心設計理念 (Design Philosophy)

初學者在面對韓文文法時，首重「錨點」。結構必須像「樂高積木」一樣規律，減輕認知負擔。

- **去文字化**：將長篇大論轉化為結構化的標籤區塊。
*   **視覺引導**：公式 (Formula) 與 禁忌 (Warnings) 必須透過 UI 特效（如背景色、框線）予以突出。
- **動態關聯**：解說中的例句應與「例句銀行」連動，而非死板的 Markdown 純文字。

---

## 2. 標準化四模組結構 (4-Module Frame)

建議所有 `.md` 解說文件應遵循以下順序編寫：

### A. 🎯 核心意義 (Definition)
- 最直覺的中文定義（例如：`~해서` = `所以` / `因此`）。
- 簡單的一句話應用場景。

### B. 📐 句型公式 (Grammar Formula)
- **視覺化公式**：`動詞詞幹 + (으)러`。
*   **收音變遷規則**：使用表格或條列式清晰列出。

### C. ⚠️ 使用禁忌與差異 (Warnings & Nuances)
- **錯誤範例**：避免初學者常見的誤用。
- **同義詞對比**：與相似語法的快速對照表（例如 `그래서` vs `니까`）。

### D. 💬 實戰語境 (Context)
- 說明什麼時候講這句話會最自然（如：職場、日常生活）。

---

## 3. UI/UX 應對實作方針

### 1. 語法公式盒 (Bento-style Formula)
- **UI 偵測**：當 Markdown 解析器偵測到 `### 📐 句型公式` 時。
- **渲染效果**：自動套用一個帶有淺藍色背景（`var(--accent)` 的低飽和度版本）與細緻虛線框的容器，並整合「點擊播放」公式音檔的能力。

### 2. 警告資訊層級 (Alert Box)
- **UI 偵測**：偵測到 `### ⚠️` 關鍵字。
- **渲染效果**：採用黃色警示背景與左側強調條，並在開頭自動插入 `! [Caution]` 警示。

### 3. 行內例句互動 (Inline Example Interactivity)
- **目標**：讓 `.md` 內文中的例句不再是死的。
- **技術方案**：利用 Regex 匹配標籤 `[ko:밥 먹으러 가요|zh:去吃飯吧]`，自動生成帶有音檔播放鈕的小卡片。
