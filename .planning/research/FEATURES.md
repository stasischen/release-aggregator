# Research: Features

## Table Stakes
- Clear startup protocol and task routing.
- Explicit execution-mode tagging (`classic_stage` / `gsd_phase`).
- Repeatable release staging with schema validation.
- Mandatory closeout evidence and worklog sync.

## Differentiators
- Wave-based multi-plan execution per phase.
- Strong traceability from requirements to phase plans and UAT.
- Cross-repo boundary enforcement with task decomposition.

## Anti-Features
- Ad-hoc cross-repo edits in a single execution step.
- Protocol decisions stored only in chat.
