# UNITFAC-002: A1/A2 Unit Skeleton Specification (Production-Ready)

## Goal / Scope

This document defines the "Production-Ready" (PR) standard for A1/A2 survival units within the Lingourmet platform. It serves as the source of truth for:
- **UNITFAC-003**: The logic for generating unit scaffolds.
- **UNITFAC-004**: The lint rules used to validate unit fixtures.
- **UNITFAC-006**: The completion of the A1-U05 pilot unit.

The goal is to ensure that all units generated by AI agents or human authors maintain a consistent educational quality, structural integrity, and learner experience.

---

## Definitions

### 1. Scaffold/Demo Fixture
A minimal unit (typically 3-5 nodes) used to demonstrate technical rendering or specific features. It lacks the full educational loop and is **not** suitable for release to learners.
- *Examples*: Current A1-U05 (v0).

### 2. Production-Ready (PR) Fixture
A comprehensive unit (typically 12-15 nodes) that satisfies all sequencing, distribution, and output progression rules defined in this spec. It is fully playable and educationally sound.
- *Examples*: A1-U04 (v0) is close, but requires minor additions to meet the PR standard.

---

## Minimum Node Skeleton (A1/A2)

A production-ready unit MUST contain the following node set, ordered by their educational role:

| Learning Role | Content Form | Min Count | Requirement |
| :--- | :--- | :--- | :--- |
| **Immersion Input** | `dialogue` | 1 | Natural context introduction. |
| **Immersion Input** | `notice` \| `message_thread` \| `comparison_card` | 1 | **Non-dialogue** real-world input. |
| **Comprehension** | `comprehension_check` (New) | 1 | Non-scored check after the first input. |
| **Structure Pattern** | `pattern_card` | 1 | Key sentence frames/formulas. |
| **Structure Grammar** | `grammar_note` | 1 | Brief explanation of "why" (e.g., counters). |
| **Structure Pattern** | `functional_phrase_pack` | 1 | Chunk-oriented dictionary/vocabulary. |
| **Controlled Output** | `practice_card` (`chunk_assembly`) | 1 | Mechanics: building sentences from chunks. |
| **Controlled Output** | `practice_card` (`response_builder`) | 1 | Context: choosing the right response. |
| **Pattern Transform** | `practice_card` \| `transform_node` (New) | 1 | Changing one variable (e.g., Ice -> Hot). |
| **Guided Output** | `roleplay_prompt` (Speaking) | 1 | Scenario-based production. |
| **Guided Output** | `message_prompt` (Writing) | 1 | Context-based production. |
| **Repair Practice** | `practice_card` (Repair) (New) | 1 | Learning to say "I don't understand" etc. |
| **Review Retrieval** | `review_card` | 1 | Final recall check (Form + Function). |
| **Transfer** | `scheduled_followup` | 2 | Spaced repetition in new contexts. |

---

## Sequencing Rules

Units follow a "Linear-Spiral" progression logic:

1.  **Input First**: At least two `immersion_input` nodes must precede any output nodes.
2.  **Checkpoint Requirement**: A `comprehension_check` must immediately follow the first major input node.
3.  **Discovery before Explanation**: Dictionary and Patterns should follow inputs to encourage discovery.
4.  **Output Scaffold**: `controlled_output` -> `pattern_transform` -> `guided_output`. This order is **strict**.
5.  **Closing Check**: `review_retrieval` must be the final node of the primary sequence.
6.  **Followups**: `scheduled_followups` are defined outside the primary sequence but must reference patterns introduced in the unit.

---

## Content Form Distribution Rules

1.  **Diversity**: A unit cannot consist only of `dialogue` and `practice_card`. It MUST include at least one "document-style" input (`notice` or `message_thread`).
2.  **Dictionary Chunks**: Dictionary packs must favor "Collocations/Chunks" (e.g., `아침을 먹다`) over "Isolated Nouns" (e.g., `아침`).
3.  **Bilingual Integrity**: Every node must have `zh-TW` translations for all learner-facing strings.

---

## Output Progression Rules (The "3-Step Production")

To prevent learner anxiety at A1/A2 levels, the output progression must follow these steps:

1.  **Step 1: Controlled (Mechanical)**
    - *Mode*: `chunk_assembly`.
    - *Goal*: Assemble the target pattern correctly without worrying about spelling.
2.  **Step 2: Transform (Productive)**
    - *Mode*: `response_builder` or `pattern_transform`.
    - *Goal*: Given Pattern A (Ice), produce Pattern B (Hot) based on a new constraint.
3.  **Step 3: Guided (Integrative)**
    - *Mode*: `guided`.
    - *Goal*: Full sentence production in a roleplay or message scenario with minimal hints.

---

## Review / Transfer Rules

1.  **Retrieval Focus**: `review_retrieval` nodes MUST be tagged with `retrieval_target`: `form`, `function`, or `mixed`.
2.  **Transfer Timing**:
    - Followup 1: `+1 unit` (Reuse the strongest pattern).
    - Followup 2: `+3 units` (Reuse a secondary pattern or repair phrase).
3.  **Pattern Persistence**: Followups must explicitly name the `transfer_pattern_refs`.

---

## Demo vs Production-ready Criteria

| Feature | Demo / Scaffold | Production-Ready (PR) |
| :--- | :--- | :--- |
| **Node Count** | < 8 | 12 - 18 |
| **Non-Dialogue Input** | Optional | **Mandatory** |
| **Comprehension Check** | Rare | **Mandatory** |
| **Transform Layer** | Missing | **Mandatory** |
| **Repair Practice** | Missing | **Mandatory** |
| **Followups** | 0-1 | 2+ with explicit targets |
| **Lint Status** | Warnings allowed | **Zero Blockers** |

## Lint/QA Rule Candidates

These rules will be implemented in `UNITFAC-004`:

### Blocker Rules (Error)
- `ERR_MIN_NODE_COUNT`: Unit has fewer than 10 nodes.
- `ERR_MISSING_ROLE`: Missing `immersion_input`, `structure_pattern`, `guided_output`, or `review_retrieval`.
- `ERR_ORDER_VIOLATION`: Output nodes appear before input/structure nodes.
- `ERR_MISSING_ZH_TW`: Any learner-facing Korean string lacks a `zh-TW` translation.
- `ERR_MISSING_COMPREHENSION`: No comprehension check found after the first input.

### Warning Rules (Warning)
- `WARN_FIXED_INPUT_TYPE`: Unit only uses `dialogue` for inputs (needs variety).
- `WARN_MISSING_REPAIR`: No repair/recovery practice node found.
- `WARN_LOW_FOLLOWUP_COUNT`: Fewer than 2 scheduled followups.
- `WARN_ISO_NOUNS`: Dictionary pack contains mostly single-word nouns without chunks.

## Mapping to Current Examples

### A1-U04 (Coffee Shop)
*   **What satisfies the spec**: Good input variety, strong chunk assembly, clear guided output, existing followups.
*   **What is missing**: 
    1. `comprehension_check` (needs to be added after U04-L1).
    2. Explicit `pattern_transform` (needs to be added after U04-P1).
    3. `repair_practice` (e.g., "Wait a moment" or "Can I pay by cash instead?").

### A1-U05 (Pharmacy)
*   **What satisfies the spec**: Nothing (it is a placeholder scaffold).
*   **What is missing**: 
    - 80% of the required nodes.
    - Diversity of inputs (only one dialogue).
    - Transform and Repair layers.
    - All followups.

## Scaffold Template Sequence (Machine-Friendly)

Recommended ordering for the `UNITFAC-003` generator:

1.  `{unit_id}-L1` (Immersion: Dialogue)
2.  `{unit_id}-L2` (Comprehension: Check)
3.  `{unit_id}-L3` (Immersion: Non-Dialogue)
4.  `{unit_id}-D1` (Structure: Dictionary Chunks)
5.  `{unit_id}-G1` (Structure: Pattern Card)
6.  `{unit_id}-G2` (Structure: Grammar Note)
7.  `{unit_id}-P1` (Output: Controlled Assembly)
8.  `{unit_id}-P2` (Output: Controlled Response)
9.  `{unit_id}-P3` (Output: Transform)
10. `{unit_id}-P4` (Output: Repair)
11. `{unit_id}-P5` (Output: Guided Production 1)
12. `{unit_id}-P6` (Output: Guided Production 2)
13. `{unit_id}-R1` (Review: Retrieval)
14. `{unit_id}-X1` (Followup: +1 Unit)
15. `{unit_id}-X2` (Followup: +3 Units)
