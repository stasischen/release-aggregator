# Knowledge Lab 內容撰寫規範 (Content Specification)

本文件定義了 Knowledge Lab (知識文庫) 內容的撰寫標準，旨在確保所有知識點（文法、句型、慣用語等）具備一致的優質視覺呈現與良好的初學者互動體驗。

## 1. 核心結構：標準化四模組 (4-Module Frame)

所有新版知識點內容建議遵循以下 Markdown 結構順序：

### A. 摘要 (Summary)
位於 JSON 的 `summary_zh_tw` 欄位。
- **目標**：用一句話說清楚這個文法是做什麼用的。
- **長度**：建議 30 字以內。

### B. 句型公式 (Formula Box) - `### 📐`
- **關鍵字**：必須使用 `### 📐` 作為標題開頭。
- **內容**：描述文法的形態變化規則。
- **範例**：
  ```markdown
  ### 📐 句型公式
  - **母音結尾**：`語幹 + ~ㄹ 거예요`
  - **子音結尾**：`語幹 + ~을 거예요`
  ```

### C. 注意事項 (Alert Box) - `### ⚠️`
- **關鍵字**：必須使用 `### ⚠️` 作為標題開頭。
- **內容**：提醒易錯點、敬語差異、常見陷阱或特殊限制。
- **視覺**：在 UI 中會以黃色警告樣式呈現。

### D. 實戰語境 (Context) - `### 💬`
- **關鍵字**：建議使用 `### 💬` 作為標題開頭。
- **內容**：說明此文法最常出現的場景（例如：日常計畫、餐廳點餐、面試等）。

---

## 2. 互動式行內例句語法 (Inline Sentence Chips)

為了增加互動性，內文中出現的韓文例句應避免使用純文字，改用以下擴充語法：

### 語法格式
`[ko:韓文原文|zh:中文翻譯|id:例句ID(可選)]`

### 範例
- **基本用法**：`[ko:갈 거예요|zh:會去]`
- **帶有音檔參考**：`[ko:갈 거예요|zh:會去|id:ex.ko.grammar.future.v1]`

### 視覺效果
- 在網頁端會呈現為一個**可點擊的藥丸形狀積木 (Sentence Chip)**。
- 點擊時會自動播放音檔（或觸發 TTS）。
- 預設顯示韓文與中文對譯，滑鼠懸停時具備高亮效果。

---

## 3. 全域例句銀行引 (Example Bank)

JSON 中的 `example_sentence_refs` 陣列應引用 `content-ko/content/core/learning_library/example_sentence/` 下的 ID。

- **原則**：詳情頁面下方的「精選例句」會自動從全域銀行中抓取。
- **重複利用**：若多個知識點共用一個例句，請使用同一個 ID，確保音檔資源不重複。

---

## 4. 最佳實踐與檢查單 (Checklist)

- [ ] 標題是否使用了正確的 Emoji 關鍵字 (📐, ⚠️)？
- [ ] 內文中的關鍵例句是否已包裹為 `[ko:...|zh:...]` 格式？
- [ ] 形態變化規則是否清晰（母音/子音/特殊音）？
- [ ] 是否已從 `example_sentence_refs` 引用了至少 2-3 個最具代表性的全域例句？

---

## 5. 向後相容性 (Backward Compatibility)
- 舊版內容若未使用 Emoji 關鍵字，檢視器將以「一般 Markdown」模式呈現，不會產生 Bento 容器外框。
- 建議優先將 A1 級別的核心文法優先升級至此新規範。
