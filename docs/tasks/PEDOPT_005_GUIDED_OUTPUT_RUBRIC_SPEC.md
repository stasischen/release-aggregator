# PEDOPT-005 — Guided Output Completion Rubric Spec

## 1. Goal
為 `guided_output`（如：引導式口說/寫作）建立生產級別的完成標準（Completion Rubric）。目標是確保作者、PM 與評測者能有一致的準則來判斷學習者的產出是否「達成教學目標」，而非僅僅符合格式要求。

---

## 2. Task Families (任務類型)

| 家族名稱 (Task Family) | 定義與場景 | 核心檢核點 |
| :--- | :--- | :--- |
| **`sentence_completion`** | **填句/補全**：給予部分框架，要求填入特定內容。 | 是否包含指定的語法/詞彙？ |
| **`constrained_response`** | **情境應答**：根據情境提示做出符合要求的回答。 | 是否符合語境且語氣合宜？ |
| **`guided_rewrite`** | **引導改寫**：將現有句子改為對應的時態或語氣。 | 變換部分是否正確且保留原意？ |
| **`role_based_output`** | **角色扮演唱戲**：在多輪對話中扮演特定角色產出內容。 | 是否順利推進對話且符合角色設定？ |

---

## 3. Rubric Structure (評分框架)

每個 `guided_output` 的 payload 應包含以下 Rubric 欄位，以指導自動化檢核（未來）與人工審閱。

### 核心欄位定義
- **`completion_conditions`**: 定義任務由「開始」進入「完成」的觸發點（如：輸入長度、錄音成功、包含關鍵詞）。
- **`required_elements`**: 必須出現的韓文關鍵詞、句型或助詞（如：必須包含 `-아요/어요`）。
- **`acceptable_variants`**: 明確說明哪些「雖然不是標準答案但正確」的變體（如：助詞省略、不同的同義詞、合法的語序變動）。

### 錯誤分類 (Error Classes)
| 分類代碼 | 描述 | 反饋策略 (Feedback Style) |
| :--- | :--- | :--- |
| **`missing`** | 遺漏核心要素。 | 指出「還少了什麼」，而非直接給答案。 |
| **`semantic_mismatch`** | 符合格式但意思與題目要求相反。 | 用提問引導：「剛才提到的是『去』還是『來』？」 |
| **`register_mismatch`** | 敬語等級（尊卑）不符合情境要求。 | 提示對象：「這是對上司說話，語氣要更客氣喔。」 |
| **`grammar_form_mismatch`** | 詞根正確但詞尾變化、時態、連接詞錯誤。 | 提示形態：「是過去的事情，句子結尾要用過去式。」 |

---

## 4. Scoring Criteria (判斷準則)

| 等級 | 準則描述 | 後續動作 |
| :--- | :--- | :--- |
| **PASS (通過)** | 包含所有 `required_elements`，語氣符合，無嚴重語意錯誤。 | 進入下一步，給予正向回饋。 |
| **REVISE (修正)** | 語意可通、核心要素具備，但有次要語法瑕疵或語氣不精準。 | 標註「小小提醒」，顯示正確參考範例但算過關。 |
| **FAIL (失敗)** | 遺漏核心要素、語意完全錯誤或無法辨識、語氣極度不合適（如對平輩用最高敬語）。 | 要求重新回答，給予 Actionable Feedback。 |

## 5. A1/A2 Examples

### 範例 1：A1 - Constrained Response (點餐數量)
*任務：在咖啡廳點「兩杯美式咖啡」*

- **Prompt**: "請告訴店員你要點兩杯美式咖啡。"
- **Required Elements**: `아메리카노` (美式咖啡), `두` (二), `잔` (杯)
- **Acceptable Variants**:
  - `아메리카노 두 잔 주세요.` (最標準)
  - `아메리카노 두 개 주세요.` (生活中可接受)
  - `아메리카노 두 잔요.` (略微省略，可接受)
- **Reject Examples**: 
  - `커피 두 잔` (太籠統，未指名美式)
  - `아메리카노 한 잔` (數量錯誤)
- **Hint Text (FAIL)**: "請確認你要點的是『美式咖啡』，而且是『兩杯』喔。"

### 範例 2：A1 - Sentence Completion (自我介紹)
*任務：完成句子介紹國籍*

- **Prompt**: "我是（台灣）人。"
- **Required Elements**: `대만` (台灣), `사람` (人), `-이에요` (是)
- **Acceptable Variants**:
  - `저는 대만 사람이에요.`
  - `대만 사람입니다.` (格式較正式，可接受)
- **Reject Examples**:
  - `대만 친구예요` (意思變成「是台灣朋友」)
- **Hint Text (FAIL)**: "請確實填入『台灣』與『人』，並加上結尾『이에요』。"

### 範例 3：A2 - Guided Rewrite (時態轉換)
*任務：將「昨天做了什麼」的回答改為過去式*

- **Prompt**: "將『친구를 만나요 (見朋友)』改為昨天發生的事。"
- **Required Elements**: `만났어요` (見了 - 過去式)
- **Acceptable Variants**:
  - `친구를 만났어요.`
  - `친구 만났어요.` (省略助詞)
- **Error Class (Register Mismatch)**: 
  - `친구를 만났습니다.`（若題目要求是 Informal Polite，此為 Register Mismatch）
- **Hint Text (Error)**: "別忘了『昨天』發生的事，結尾要變成『-았어요/었어요』喔。"

### 範例 4：A2 - Constrained Response (拒絕約會)
*任務：婉拒朋友的週六邀約，理由是「有工作」*

- **Prompt**: "週六有工作，請委婉地拒絕朋友的邀約。"
- **Required Elements**: `토요일` (週六), `일이 있어요` (有工作/事情), `미안해요` 或 `안 돼요` (抱歉/不方便)
- **Acceptable Variants**:
  - `미안해요. 토요일에 일이 있어요.`
  - `토요일에 일이 있어서 못 가요.`
- **Reject Examples**:
  - `싫어요` (韓文語境中太直接，判定為 Register/Style Fail)
- **Hint Text (Semantic/Style)**: "拒絕朋友時要先表達歉意，並敘述『工作』這個理由喔。"

## 6. PM Checklist & Mapping (審核清單)

PM 在 Review 單元時應檢查：
1. **[ ] 邊界清晰**：`required_elements` 是否足夠具體（不會讓對的答案被判錯）？
2. **[ ] 難度分級**：A1 是否避免了複雜的語氣檢核？A2 是否開始要求時態與基本語氣一致？
3. **[ ] 反饋可用性**：`feedback_zh_tw` 是否「短小、好改、不囉唆」？（避免說「你錯了」，要說「請把結尾改成...」）。
4. **[ ] 變體寬容度**：是否考慮了常見的助詞省略（如 `을/를` 省略在口說中是否算對）？

---

## 7. Consistency Note
本規格與 `PEDOPT-001~004` 保持一致：
- 使用 `zh_tw` 作為反饋與提示語言。
- 區分 `A1/A2` 教學目標密度。
- 教育重心放在「生存溝通」與「遷移能力」。
