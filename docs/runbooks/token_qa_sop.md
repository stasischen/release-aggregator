# Token Candidates 100% QA SOP

This document provides a **QA gate** procedure for `content-ko`. Use this for verifying tokenizations and flagging polysemy after content updates or rule changes.

## Prerequisites

- Repository: `content-ko`
- Python 3.10+
- `token_candidates.jsonl` (generated by `run_mapping_pipeline.py`)

---

## Phase 1: Regenerate Token Candidates

If rules or engine code changed, regenerate tokens with a clean cache:

1. **Clear the token cache** (PowerShell):

```powershell
Remove-Item data\cache\* -Recurse -Force
```

2. **Run the mapping pipeline**:

```powershell
python scripts/ops/run_mapping_pipeline.py
```

3. **Verify completion**: Check for `✅ Finished Phase 5 Run` in output.

---

## Phase 2: Run Automated QA Audit

4. **Run the comprehensive audit script**:

```powershell
python scripts/dev/audit_token_qa.py
```

5. **Review the audit output**:

| Check | Gate Threshold | Action if Failed |
| :--- | :--- | :--- |
| **Verb Ending Leaks** | Must be **0** | Fix rules in `engine/rules/30_verb_endings.json` |
| **Fallback Tokens** | Review each one | Add to `content/mappings/` or write a rules |
| **Low Confidence** | Review if < 0.5 | Add to mapping or use `dictionary_bypass: true` |
| **Missing Dict** | Informational | Add entries to `content/mappings/dict_ko_zh_tw.json` |
| **Polysemy** | Review list | Add `#sense:surface=index` tag in Yarn |

---

## Phase 3: Fixing Issues

- **Verb Ending Leaks**: Raise priority in `engine/rules/30_verb_endings.json` above 95.
- **Fallback/Unknown**: Add to corresponding `content/mappings/mapping_{type}.json`.
- **Dictionary**: Add translations to `content/mappings/dict_ko_zh_tw.json`.
- **Polysemy**: Add `#sense:surface=sense_index` tag to Yarn files.

---

## Quick Reference: File Locations

| File | Purpose |
| :--- | :--- |
| `scripts/ops/run_mapping_pipeline.py` | Pipeline runner |
| `scripts/dev/audit_token_qa.py` | QA gate script |
| `content/mappings/` | Split mapping files (nouns, verbs, etc.) |
| `content/mappings/dict_ko_zh_tw.json` | Main dictionary |
| `data/staging/lessons/` | Per-lesson token output |
